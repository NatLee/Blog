4. 資料分析

最後，我們可以用 describe() 去看整體資料的分布：

看到這邊，有些人可能會有疑問。因為 ES 的限制是最多一次取出 10000 筆資料，為什麼 Eland 可以超過上限？

這邊有一篇官方論壇的QA有提到 ES 限制的事 max-limit-for-number-of-search-results

實際上，Eland 的 code 裡面相關程式碼 etl.py#L233 顯示它會一萬筆一萬筆循序把資料全部讀取出來。

也就是說，它只要呼叫一次函數就會使用多次讀取達到超過 10000 筆上限的限制！
